## この本は何について

この本では、あなた自身の手で **React をゼロから再発明** します。

React の API の使い方を学ぶ本ではありません。**React が生まれるまでの進化の道のりを、もう一度たどる本** です。最も原始的な `document.createElement` から始めて、ひとつずつ痛みを体験し、その痛みから自然に解決策を導きます。最後には、それらの解決策こそが React の核になる設計思想だとわかります。

各章は同じ流れです。**まず痛みを感じる。次に解決策を見つける。最後に自分の手で実装する。** 読み終えたとき、あなたの手元には約 **400 行** の mini-react が残ります。小さいですが、Virtual DOM、Time Slicing、Fiber Reconciliation、同期 Commit、そして Hooks の核心をきちんと備えています。

## 執筆スタイル：ソクラテス式対話

この本はすべて **対話形式** で進みます。登場人物は二人です。

- **Shifu (🧙‍♂️)**: フロントエンドの進化をよく知る師匠。答えをそのまま渡しません。問いを重ねて、Po に自分で答えを見つけさせます。
- **Po (🐼)**: React はまだ知らないけれど、基礎はある。好奇心が強く、いい質問を投げる熊猫です。

## この本は誰向けか

この本は、JavaScript、HTML、DOM 操作の基本を知っている人を想定しています。

React を触ったことがない人でも、フロントエンドのフレームワークをほとんど使ったことがない人でもかまいません。ゼロから React のはっきりした心の地図を作りたい人に向いています。

仕事で React をたくさん使っているけれど、中身はあまり知らない。そんな人がブラックボックスの中をのぞきたいときにも役に立ちます。
